# Find Help - מחקר בלי לנחש

ל-AI יש תאריך עצירה. כלים משתנים. לכן בודקים לפני שמבטיחים.

הסבר לילד: "תיעוד (documentation) הוא מדריך ההפעלה של כלי."

## מתי לעצור ולבדוק

- לפני ספרייה, API, שירות ענן, התחברות, מפה, מצלמה, או AI חיצוני.
- אחרי אותו באג פעמיים.
- כשצריך דוגמה פשוטה מקוד פתוח.
- כשהתשובה תלויה במה שקיים היום.

## איך בודקים

1. חפש תיעוד רשמי קודם.
2. בדוק גרסה מותקנת אם יש פרויקט עם `package.json`.
3. חפש דוגמה קטנה ב-GitHub רק אם היא עוזרת להבין.
4. הסבר לילד מה מצאת בשני משפטים.
5. בחר את הדרך הכי פשוטה ובטוחה.

## איך מסבירים קוד פתוח

אמור:

```
קוד פתוח הוא פרויקט שאנשים שיתפו כדי שאחרים ילמדו ממנו.
אנחנו לא מעתיקים בלי להבין. קודם קוראים רעיון קטן.
```

## גבולות לילדים

- לא מכניסים מפתחות API או סיסמאות.
- לא מבקשים מהילד להירשם לשירות בלי הורה.
- לא מתקינים חבילה אם HTML פשוט מספיק.

## בדיקת סיום

סיים עם:

```text
מה בדקתי?
מה הדרך הפשוטה?
מה הילד יכול לנסות עכשיו בדפדפן?
```
