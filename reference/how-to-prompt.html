איך כותבים מפרט

עובד בכל סוכן קוד - Cursor · Claude Code · Codex · Copilot · Gemini

מפרט = מה שאתם אומרים לסוכן לפני שהוא בונה. קודם רעיון ותכנון - ראו המסע המלא.

לפני המפרט

רעיון: "תן 3 רעיונות - אל תכתוב קוד"

תכנון: מי ישתמש? מה קורה כשמצליחים?

מפרט טוב = שלושה חלקים

1. מטרה - מה לבנות?

"משחק נחש. חצים מזיזים. תפוחים = ניקוד. פגיעה בזנב = הפסד."

2. תחום - כמה לשנות?

שינוי אחד בכל פעם - לא עשר בקשות בבת אחת. "בלי ספריות חיצוניות."

3. בדיקה - איך יודעים שזה עובד?

"כשפותחים בדפדפן: הנחש זז, ניקוד למעלה, המשחק נגמר בפגיעה."

לא צריך "לתייג" קבצים. הסוכן כבר רואה את הפרויקט שפתחתם - הוא יבחר את הקובץ הנכון לבד. פשוט תארו מה אתם רוצים.

דוגמה שלמה

תבנה משחק נחש בפרויקט.
מטרה: חצים מזיזים, תפוחים מוסיפים ניקוד, פגיעה בזנב = הפסד.
תחום: שינוי אחד, בלי ספריות.
בדיקה: בדפדפן רואים נחש, ניקוד, וסוף משחק כשפוגעים בעצמך.
תסביר בפשטות איך לבדוק.

חלש מול חזק

חלש: "תעשה משחק מגניב"

הסוכן לא יודע מה אתם רוצים. התוצאה אקראית.

חזק: שלושת החלקים למעלה - ברור, אפשר לבדוק.

משפטים שתמיד עובדים

  • "תסביר בפשטות למה ___"
  • "קורה ___, רציתי ___. תתקן רק את זה."
  • "תן לי רמז, אל תפתור בשבילי"
  • "מה הצעד הקטן הבא?"
