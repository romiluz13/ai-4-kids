# Kid Build Workflow — שבעת השלבים

תהליך מלא לבנייה עם סוכן קוד. **אל תדלג על שלבים.**

## השלבים

| # | שם | מה | קובץ |
|---|-----|-----|------|
| 1 | רעיון | 3 רעיונות, הילד בוחר | שיחה — `kid-brainstorm` |
| 2 | תכנון | מי משחק? מה קורה? מה לא? | שיחה קצרה |
| 3 | מפרט | משפט אחד + 4 חלקי פרומפט | `templates/SPEC.md` |
| 4 | צעדים | 2–4 משימות קטנות | `templates/TASKS.md` |
| 5 | בנייה | משימה **אחת** — עריכת `game.html` | Agent mode |
| 6 | בדיקה | פתיחה בדפדפן — האם עובד? | הילד מאשר |
| 7 | שיפור | תיקון אחד | חזרה ל-6 |

## מצבים לפי כלי

- **רעיון / תכנון / מפרט** → Ask (בלי לשנות קבצים)
- **צעדים** → Plan או Ask
- **בנייה** → Agent
- **בדיקה** → הילד בדפדפן; אתה מנחה מה לבדוק

## אם הילד ממהר

"אני מבין שאתה רוצה לראות משהו מהר. בואו נבחר רעיון בדקה אחת — ואז בונים."

## אחרי כל בנייה

תמיד: "פתח את game.html בדפדפן ובדוק: [מה שכתוב במפרט]."
