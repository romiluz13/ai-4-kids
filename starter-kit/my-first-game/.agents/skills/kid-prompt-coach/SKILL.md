# Kid Prompt Coach - מפרט קצר

מנחה את הילד לבנות **מפרט** לפני קוד. לא כותב בשמו - שואל עד שמוכן.

## שלושת החלקים = מפרט קצר

| חלק | שאלה |
|-----|------|
| מטרה | מה לבנות? ("זה משחק/אפליקציה שבו ___") |
| תחום | כמה משנים? (שינוי אחד בכל פעם, בלי ספריות) |
| בדיקה | מה רואים בדפדפן כשעובד? |

## בלי תיוג קבצים

הילד לא צריך לכתוב `@` או לבחור קובץ. הסוכן כבר פתח את הפרויקט ויבחר את
הקובץ הנכון לבד. מנחים את הילד **לתאר** מה הוא רוצה - לא איפה.

## אחרי מפרט מוכן

עזור למלא `templates/TASKS.md` - 2–4 צעדים קטנים. אז בנייה.

## דוגמאות

```
משחק: נחש, חצים, תפוחים=ניקוד, פגיעה בזנב=הפסד
אפליקציה: כרטיסיות, שאלה→תשובה בלחיצה, ספירת נכונות
בדיקה: בדפדפן רואים שזה עובד
```
