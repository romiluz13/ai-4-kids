# Kid Build Workflow — שבעת השלבים

תהליך מלא לבנייה עם סוכן קוד. **אל תדלג על שלבים.**

## השלבים

| # | שם | מה | קובץ |
|---|-----|-----|------|
| 1 | רעיון | 3 רעיונות, הילד בוחר | `kid-brainstorm` |
| 2 | תכנון | מי משתמש? מה קורה? מה לא? | שיחה קצרה |
| 3 | מפרט | משפט אחד + 4 חלקים | `templates/SPEC.md` |
| 4 | צעדים | 2–4 משימות קטנות | `templates/TASKS.md` |
| 5 | בנייה | משימה **אחת** | `ai-build-buddy` |
| 6 | בדיקה | פתיחה בדפדפן | הילד מאשר |
| 7 | שיפור | תיקון אחד | חזרה ל-6 |

## רמה 1 — ניצחון מהיר קודם

אם הילד חדש לגמרי: הבא תוצאה לפעול על המסך מהר, ואז הסבר. אל תיתקע בתכנון ארוך.

## מצבים לפי כלי

- **רעיון / תכנון / מפרט** → Ask (בלי לשנות קבצים)
- **צעדים** → Plan או Ask
- **בנייה** → Agent
- **בדיקה** → הילד בדפדפן; אתה מנחה מה לבדוק

## אם הילד ממהר

"אני מבין שבא לך לראות משהו מהר. נבחר רעיון בדקה — ואז בונים."

## אחרי כל בנייה

תמיד: "פתח בדפדפן ובדוק: [מה שכתוב במפרט]."
