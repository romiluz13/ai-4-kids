איך עובדים היום

המורה — מחשב + מסך גדול + סוכן קוד (Cursor / Claude Code / Codex / ועוד)
אתם — מחשב עם אינטרנט. בכיתה אין סוכן קוד אצלכם — וזה בסדר.

בבית — עם הורים: תשבטו את המאגר ותתחילו עם סוכן אמיתי.

מה זה סוכן קוד?

סוכן קוד = AI שמחובר לתיקיית פרויקט על המחשב.

  • קורא וכותב קבצים
  • מסביר, מחפש בפרויקט, לפעמים ברשת
  • Cursor · Claude Code · Codex · Copilot — אותו רעיון, אפליקציות שונות

זה לא צ'אט רגיל. הצ'אט מדבר. הסוכן בונה בתיקייה.

מי מחליט מה לבנות?

יסודות קוד — מה חייבים להבין

קובץ — משהו שנשמר (game.html)

תיקייה — מארגנת קבצים

פרויקט = תיקייה אחת

דפדפן — שם רואים ובודקים משחק

באג — משהו שלא עובד. נורמלי.

Open Folder — פותחים את תיקיית הפרויקט בסוכן, לא קובץ בודד

התהליך — איך מקצוענים עובדים עם AI

לא קופצים ישר לקוד. יש שבעה שלבים:

1 רעיון2 תכנון3 מפרט4 צעדים 5 בנייה6 בדיקה7 שיפור
  1. רעיון — 3 רעיונות, בוחרים אחד. בלי קוד.
  2. תכנון — מי משחק? מה קורה כשמנצחים?
  3. מפרט — "זה משחק שבו ___" + מה בודקים בדפדפן
  4. צעדים — 2–4 משימות קטנות, אחת בכל פעם
  5. בנייה — הסוכן משנה קבצים (על המסך הגדול היום)
  6. בדיקה — פותחים בדפדפן. אתם הבוס — AI טועה לפעמים
  7. שיפור — "קורה X, רציתי Y" — תיקון אחד

מה נבנה ביחד — על המסך הגדול

  1. סיבוב רעיון — 3 רעיונות, אתם בוחרים
  2. בונים מפרט ביחד (מטרה · תחום · בדיקה)
  3. המורה מריץ סוכן → רואים קובץ משתנה → דפדפן
  4. מראים SKILL.md בתוך הפרויקט
  5. אתם בוחרים תיקון אחד — שוב על המסך

מה אתם עושים על המחשב שלכם — היום

אין לכם סוכן בכיתה. אתם עושים את החלק שלכם בתהליך:

  • רעיון — חפשו באינטרנט השראה למשחקים פשוטים
  • מפרט — מלאו תבנית על הנייר / במסמך (המורה יחלק)
  • הצבעה — איזה רעיון? איזה תיקון?
  • בדיקה — כשהמשחק על המסך: "עובד? מה חסר?"
  • גלישה במאגר — פתחו את GitHub — README, המסע המלא
